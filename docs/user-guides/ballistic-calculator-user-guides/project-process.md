# Процесс изменений и этапы проекта

## Где смотреть текущий этап

- [PROJECT_STATUS.md](https://github.com/Vladimir2517/ballistic-calculator/blob/main/PROJECT_STATUS.md)

## Где смотреть историю изменений

- [CHANGELOG.md](https://github.com/Vladimir2517/ballistic-calculator/blob/main/CHANGELOG.md)

## Базовый workflow

1. Создать Issue (bug/feature).
2. Обновить `PROJECT_STATUS.md` перед началом правок.
3. Сделать ветку `feature/...` или `fix/...`.
4. Коммитить в формате Conventional Commits.
5. Открыть PR по шаблону, обновить `CHANGELOG.md`.
6. После merge перенести задачу в Done в `PROJECT_STATUS.md`.

## Зачем это нужно

Так всегда видно:
- что планируется,
- что в работе,
- что уже сделано,
- какие изменения вошли в каждую версию.
